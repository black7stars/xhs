# Skill: image-validator

## 用途

对 food-image-core 的输出进行强制性后验检查，确保配图建议和Banana提示词符合规范，未通过检查则拦截生图流程。

## 调用方式

```
skill: image-validator
输入: [food-image-core生成的完整输出内容]
```

## 检查流程

### 第一步：结构化标记检查

**检查清单**：
- [ ] 是否包含 `<!-- IMAGE_META:validation -->` 标记块？
- [ ] 是否包含 `<!-- PLATING:6items -->` 标记块？
- [ ] 是否包含 `<!-- BANANA_PROMPT:json -->` 标记块？
- [ ] 每个标记块是否正确闭合（`<!-- END_XXX -->`）？

**失败处理**：
- 缺少任何标记块 → 返回失败，要求补充标记
- 标记块未闭合 → 返回失败，要求检查格式

### 第二步：配图建议完整性检查

**检查清单**：
- [ ] PLATING块中6项是否全部勾选 `[x]`？（未勾选`[ ]`即失败）
- [ ] 盘子选择：是否有具体内容（材质+形状+尺寸+边沿）？
- [ ] 摆盘逻辑：是否有具体内容（构图法则+视觉流动）？
- [ ] 间距控制：是否包含具体cm数值（如"2-3cm"）？
- [ ] 装饰法则：是否包含四要素（装饰物+数量+分布+效果）？
- [ ] 留白艺术：是否包含具体比例（如"左侧1/3"）？
- [ ] 整体风格：是否包含对标标准（如"fine dining标准"）？

**失败处理**：
- 任何一项未勾选 → 返回失败，要求完成勾选
- 任何一项内容为空或"待定" → 返回失败，要求补充内容
- 间距缺少cm数值 → 返回失败，要求添加具体数值
- 装饰法则缺少四要素 → 返回失败，要求补充完整

### 第三步：Banana提示词格式检查

**检查清单**：
- [ ] BANANA_PROMPT块是否为合法JSON格式？
- [ ] 是否包含必需的4个字段？
  - model（必须为"nano-banana-pro"）
  - prompt（必须以"masterpiece, best quality"开头）
  - aspectRatio（必须为"3:4"）
  - imageSize（必须为"2K"）
- [ ] 是否**不含**Midjourney参数？
  - 检查：`--ar`, `--v`, `--q`, `--style`, `--s`, `--c`, `--niji` 等
- [ ] prompt是否以 `masterpiece, best quality` 开头？
- [ ] prompt是否包含摆盘6要素的英文转换？
- [ ] prompt中是否不含中文？

**失败处理**：
- JSON格式不合法 → 返回失败，要求修正为合法JSON
- 缺少必需字段 → 返回失败，要求补充字段（model/prompt/aspectRatio/imageSize）
- aspectRatio不为"3:4" → 返回失败，要求修正为"3:4"
- imageSize不为"2K" → 返回失败，要求修正为"2K"
- 发现Midjourney参数 → 返回失败，要求移除所有`--`参数
- prompt不以质量标签开头 → 返回失败，要求添加
- 发现中文残留 → 返回失败，要求转换为英文

### 第四步：素材引用检查

**检查清单**：
- [ ] 配图建议是否引用素材库YAML？（检查`素材库/图片素材库/`路径）
- [ ] 引用的素材路径格式是否正确？
- [ ] （可选）素材文件是否存在于素材库？

**处理方式**：
- 未引用素材 → 返回警告，建议检查是否应引用素材
- 路径格式错误 → 返回警告，建议修正路径格式

### 第五步：批次一致性检查（新增）

**检查清单**：

**Type A 批次（多菜品）**：
- [ ] **盘子选择**：所有配图的盘子描述是否**完全相同**？
  - 检查：材质、形状、尺寸、颜色
  - 示例：都是"纯白哑光圆形盘，直径26cm"
- [ ] **整体风格**：所有配图的风格描述是否**完全相同**？
  - 检查：风格定位词（Bistro/高端西餐）
- [ ] **背景**：所有配图的背景描述是否**完全相同**？
  - 检查：木质桌面/大理石/纯色
- [ ] **拍摄视角**：所有配图的视角是否**完全相同**？
  - 禁止：配图1俯拍，配图2用45度
- [ ] **光线**：所有配图的光线是否**完全相同**？
  - 禁止：配图1暖光，配图2冷光

**Type B 批次（单菜品多视角）**：
- [ ] **盘子选择**：所有配图的盘子描述是否**完全相同**？
- [ ] **整体风格**：所有配图的风格描述是否**完全相同**？
- [ ] **背景**：所有配图的背景描述是否**完全相同**？
- [ ] **拍摄视角**：是否按序列变化？
  - 期望序列：俯拍 → 45度 → 平视 → 特写
  - 检查：每个配图的视角是否符合序列
- [ ] **光线**：是否按序列变化？
  - 期望序列：自然光 → 暖光 → 侧光 → 顶光
  - 检查：每个配图的光线是否符合序列

**失败处理**：
- Type A 批次中任何锁定字段不一致 → 返回失败，要求统一
- Type B 批次中锁定字段不一致 → 返回失败，要求统一
- Type B 批次中变化字段未按序列 → 返回警告，建议调整

**检查方法**：
1. 提取每个配图的"摆盘说明"部分
2. 对比"盘子选择"、"整体风格"、"背景"字段
3. Type A：三者必须完全相同
4. Type B：盘子/风格/背景必须相同，视角/光线按序列变化

### 第六步：歧义词汇检查（翻译后验）

**检查方法**：
1. 将prompt中的英文关键词翻译成中文
2. 检查翻译结果是否可能引发歧义
3. 特别关注以下易歧义词汇：
   - `eye-level shot` → 易被理解为"眼睛特写"，应改为"table-level perspective"
   - `straight on view` → 易被理解为"正面肖像"，应改为"frontal flat angle"
   - `bistro` → 国外AI理解为"法式廉价小酒馆"，应改为：Modern Chinese 'Bistro' (NOT the traditional French cheap-eats tavern; reimagined as: Yunnan/Guizhou/Sichuan fusion plates + biodynamic wine + urban refuge aesthetic, 80-150 RMB per capita, targeting millennials seeking 'mountain vibes' in city centers)
   - `cozy atmosphere` → 情感模糊词，应改为具体描述

**失败处理**：
- 发现歧义词汇 → 返回警告，建议替换为更精确的表达

## 使用示例

**输入**：
```
skill: image-validator
输入：
<!-- IMAGE_META:validation -->
配图建议完整性: true
素材匹配确认: true
<!-- END_IMAGE_META -->

### 配图1
- **素材**：素材库/图片素材库/海鲜类/绯焰金鲷_001.yaml
- **菜品名**：春日绯红蟹肉塔
...

<!-- PLATING:6items -->
- [x] 盘子选择: 纯白哑光圆形瓷盘，直径26cm，无边沿
- [x] 摆盘逻辑: 黄金分割，右上到左下对角线流动
- [ ] 间距控制: 塔壳间距适中
...
<!-- END_PLATING -->

<!-- BANANA_PROMPT:json -->
{
  "model": "nano-banana-pro",
  "prompt": "masterpiece, best quality, professional food presentation, modern Chinese 'Bistro' (NOT traditional French tavern; reimagined as: Yunnan/Guizhou/Sichuan fusion plates + biodynamic wine + urban refuge aesthetic), 2k uhd, [主体描述], [摆盘要素], editorial food photography",
  "aspectRatio": "3:4",
  "imageSize": "2K",
  "shutProgress": false
}
<!-- END_BANANA_PROMPT -->
```

**输出**：
```markdown
## 生图质量检查报告

### 检查结果：❌失败

**总体评分**：75/100

### 详细检查项

#### 1. 结构化标记 ✅通过
...

#### 2. 配图建议完整性 ❌失败
- 间距数值检查：❌失败（缺少具体cm数值，当前为"适中"）
- 6项全部勾选：❌失败（间距控制未勾选）

#### 3. Banana提示词格式 ✅通过
...

### 问题详情

**严重问题（必须修改）**：
1. 间距控制缺少具体cm数值 → PLATING块第3项 → 改为"塔壳间距2-3cm"
2. 间距控制未勾选 → PLATING块第3项 → 完成勾选 [x]

### 最终判定

- **是否通过**：否
- **是否可生图**：否
- **需要返工**：是
- **返工重点**：补充间距具体数值（2-3cm）并完成勾选
```

## 注意事项

1. **强制拦截**：任何严重问题都导致整体失败，禁止继续生图
2. **明确位置**：必须指出具体标记块和项目
3. **提供建议**：不仅指出问题，还要提供具体修改方案
4. **检查顺序**：必须按1→2→3→4→5→6顺序检查，前置失败则停止后续检查
5. **返回格式**：严格按照"生图质量检查报告"格式输出
