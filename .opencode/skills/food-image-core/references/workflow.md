# 工作流定义

## 主工作流：完整生图流程

1. 接收输入（文字或图片）
2. 需求解析 / 图片分析
3. 素材匹配
4. 配图建议生成
5. 提示词转换
6. API调用生图
7. 保存结果
8. 输出简报

## 子工作流1：需求解析

**输入**：用户自然语言描述  
**输出**：结构化需求对象

解析维度：
- 食材类型：海鲜类/肉类/蔬菜类/菌菇类/其他
- 色调偏好：暖色调/冷色调/中性色
- 视角要求：俯拍/45度斜俯拍/平视/特写
- 风格定位：Bistro轻正餐/高端西餐/融合料理
- 摆盘要素：留白比例、装饰复杂度、盘子类型
- 场景/季节：春季/夏季/秋季/冬季

## 子工作流2：素材匹配

**输入**：结构化需求  
**输出**：排序后的素材列表

**匹配优先级**：
1. 食材类型（权重40%）
2. 色调偏好（权重25%）
3. 场景/季节（权重20%）
4. 摆盘风格（权重15%）

## 子工作流3：配图建议生成

**输入**：匹配素材的YAML数据  
**输出**：标准化配图建议

按「配图规范详解.md」格式生成，包含6项摆盘说明。

## 子工作流4：提示词转换

**输入**：配图建议文档  
**输出**：Banana提示词

**转换核心**：摆盘6要素 → 英文关键词

## 子工作流5：API调用

**输入**：提示词列表  
**输出**：生成的图片文件

**重试机制**：
- 最大重试：3次
- 首次失败：CFG 7.5→6.5, Steps 30→35
- 二次失败：CFG 6.5→6.0, Steps 35→40，简化提示词
- 三次失败：标记失败，保存简化提示词

## 子工作流6：图片分析

**输入**：用户上传图片  
**输出**：YAML标签 + 自动保存至素材库

调用SiliconFlow API (Kimi-K2.5 Vision) 分析图片内容。

## 输出规范

### 文件命名
- 生成的图片：`配图1.jpg`, `配图2.jpg`, ...
- 生成记录：`生成记录.md`
- 失败提示词：`配图X_待生成.txt`

### 保存路径
```
AI生图/
├── 文生图/
│   └── [日期]/
│       ├── 配图1.jpg
│       └── 生成记录.md
└── 图生图/
    └── [日期]/
        ├── 配图1.jpg
        ├── 原始图片_001.jpg
        └── 生成记录.md
```
