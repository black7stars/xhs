# Image Migration Agent

执行完整的8步图片素材迁移流程，从环境检查到自动清理。

## 职责

作为工作流控制器，协调完成以下任务：
- 第1步：环境检查
- 第2-6步：图片处理（分析→命名→分类→YAML→归档）
- 第7步：自查修正
- 第8步：自动清理

## 输入参数

```yaml
source_dir: string (源目录，默认 "参考图/")
target_dir: string (目标目录，默认 "素材库/图片素材库/")
```

## 8步工作流程

### 第1步：环境检查（10秒）

**检查项**：
- Python虚拟环境是否正常
- API密钥配置（SiliconFlow）
- 目录权限（source_dir 和 target_dir）

**输出**：环境状态报告

### 第2步：图片扫描与分析（每张约60-90秒）

**任务**：
1. 扫描 source_dir 获取图片列表
2. 按文件名排序
3. 调用 SiliconFlow API（Kimi-K2.5模型）分析每张图片：
   - 识别菜品主体、食材
   - 分析摆盘结构、色彩、光线
   - 输出分析结果

**输出**：分析结果列表

### 第3步：创意命名

**命名规范**：季节 + 颜色 + 食材 + 呈现

**示例**：
- 紫岸蛤语
- 春涧贝语
- 琥珀牛肋
- 绯红雪脍

### 第4步：自动分类

**分类规则**：
- 海鲜类
- 肉类
- 蔬菜类
- 菌菇类
- 其他

### 第5步：生成YAML

**必须包含6要素**：
1. 盘子选择
2. 摆盘逻辑
3. 间距控制
4. 装饰法则
5. 留白艺术
6. 整体风格

### 第6步：文件归档

**操作**：
1. 复制图片到对应分类目录
2. 自动生成配套YAML文件
3. 处理重名（自动添加序号_001, _002）

### 第7步：自查修正

**检查项**：
- 重复文件清理（保留最早版本）
- YAML与图片匹配检查
- 数量核对（YAML数量 = 图片数量）
- 质量校验（YAML格式、必填字段）

### 第8步：自动清理

**操作**：
1. 清空 source_dir（保留空目录）
2. 删除临时日志
3. 生成处理报告

## 输出格式

详见 command/图片素材迁移.md 的输出示例。

## 错误处理

| 错误场景 | 处理方式 |
|---------|---------|
| 环境检查失败 | 提示用户修复环境问题 |
| 图片分析失败 | 记录失败，继续下一张，支持断点续传 |
| 重名冲突 | 自动添加序号 |
| 自查发现问题 | 自动修复，生成修复报告 |
