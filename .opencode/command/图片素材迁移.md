# 图片素材迁移

将参考图中的菜品图片自动分析、创意命名、分类归档并迁移到素材库，全程自动完成包括自查修正和清理工作。

## 使用方法（只需2步）

**第1步：上传图片**
将菜品图片文件拖入 `参考图/` 目录

**第2步：运行命令**
```
/图片素材迁移
```

## 前置条件

- ✅ 支持格式：jpg、png、jpeg
- ✅ 图片建议：清晰的菜品成品图，光线良好
- ✅ 文件命名：任意命名（脚本会自动重命名）

## 自动执行流程

### 1️⃣ 环境检查（10秒）
- 检查Python虚拟环境
- 验证API密钥配置
- 确认目录权限

### 2️⃣ 图片扫描（5秒）
- 扫描`参考图/`目录
- 统计待处理图片数量
- 按文件名数字排序

### 3️⃣ AI分析与迁移（每张约60-90秒）

**智能分析阶段**：
- 调用SiliconFlow API（Kimi-K2.5模型）
- 识别菜品主体、食材、色彩、摆盘风格
- 基于分析结果生成创意命名

**自动处理阶段**：
- 🏷️ **创意命名**：使用「季节+颜色+食材+呈现」规范
  - 示例：紫岸蛤语、春涧贝语、琥珀牛肋
- 📂 **自动分类**：根据主要食材分类
  - 海鲜类 / 肉类 / 蔬菜类 / 菌菇类 / 其他
- 📝 **生成YAML**：提取6要素摆盘说明
  - 盘子选择 / 摆盘逻辑 / 间距控制 / 装饰法则 / 留白艺术 / 整体风格
- 📁 **文件归档**：复制图片到对应分类目录
- 🔢 **自动编号**：处理重名，自动添加序号（_001, _002）

**批次处理**：
- 每批处理10张图片
- 失败自动重试3次
- 支持断点续传（中断后从上次继续）

### 4️⃣ 自查修正（自动执行）

#### 检查1：重复文件清理
- 检测同一菜品的多个版本
- 保留最早版本，删除重复
- 处理编号冲突（如_001, _002）

#### 检查2：YAML与图片匹配
- 确保每个YAML都有对应图片
- 确保每个图片都有对应YAML
- 删除不匹配的孤立文件

#### 检查3：数量核对
- 验证YAML数量 = 图片数量
- 生成详细核对报告

#### 检查4：质量校验
- 验证YAML格式正确
- 检查必填字段完整
- 确认分类准确

### 5️⃣ 自动清理（10秒）

**清理工作**：
- 🧹 清空`参考图/`目录（保留空目录供下次使用）
- 🗑️ 删除临时日志文件
- 📝 生成处理报告
- 💾 保存处理记录

## 处理时间

- **1-10张**：约10-15分钟
- **10-50张**：约1-2小时
- **50-100张**：约2-3小时

> 💡 提示：建议每次上传不超过30张，分批处理更稳定

## 故障排查

**Q：命令运行失败怎么办？**
```
检查项：
1. 确认已上传图片到 参考图/ 目录
2. 确认API密钥配置正确（.env文件）
3. 确认虚拟环境正常（venv/目录）
```

**Q：部分图片处理失败？**
```
• 失败图片会自动记录到失败列表
• 检查日志了解具体原因
• 可单独重新上传失败的图片再次运行
```

**Q：处理被中断？**
```
• 命令支持断点续传
• 重新运行会自动从断点继续
• 不会重复处理已完成的图片
```

**Q：误删了文件？**
```
• 所有操作都有日志记录
• 查看日志了解删除详情
• 从原始备份恢复（如有）
```
