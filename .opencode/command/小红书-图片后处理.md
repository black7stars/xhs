---
description: 小红书笔记图片后处理修复
agent: image-post-process
---

识别用户后处理意图，执行图片修复、替换、重生成等操作。

**参数解析规则**：
- $1, $2, ...: 用户后处理指令（自然语言）
- context: 从对话上下文中提取当前笔记信息

**参数提取逻辑**：
```
# 合并所有参数作为用户指令
user_request = 合并 $1 到末尾

# 从上下文提取当前笔记信息
if 上下文中有最近生成的笔记:
    target_note = 上下文中的笔记标题
else:
    提示用户提供笔记标题或先执行 /小红书 命令
    return
```

**支持的指令类型**：
| 用户输入示例 | 操作 |
|------------|------|
| "配图2换一张" | 重新匹配素材 → 生成新图 |
| "重新生成配图3" | 使用原提示词重新调用API |
| "配图1要更多留白" | 修改摆盘说明 → 更新提示词 → 重新生成 |
| "全部重新生成" | 重新跑第9-12步 |
| "增加2张配图" | 补充配图规划 → 生成新增配图 |
| "删掉配图4" | 删除文件并更新笔记 |
| "配图5颜色不对" | 调整色调参数 → 重新生成 |

**执行流程**：
1. 识别用户意图（解析指令类型和目标配图）
2. 从上下文定位目标笔记
3. 调用 agent: image-post-process
4. 传递参数：
   - user_request: [用户指令]
   - context: [对话上下文]
   - target_note: [目标笔记标题]

**强制要求**：
- agent 必须识别具体指令类型
- 执行相应修复操作（换素材/重生成/调摆盘/改色调/增删）
- 更新笔记.md 文件同步修改

**使用示例**：
```
/小红书-图片后处理
配图2换一张

/小红书-图片后处理
重新生成配图3

/小红书-图片后处理
配图1要更多留白
```

**输出结果**：
```
✅ 修复完成

📁 修改位置: 笔记/[target_note]/
🔧 执行操作: [具体操作描述]
⏱️ 耗时: XX秒

**修改后结果**:
- [如果是换素材/调摆盘] 配图X已更新，新素材: [路径]
- [如果是重生成] 配图X已重新生成
- [如果是增删] 配图数量现为X张

📋 更新内容已同步至 笔记.md

📌 是否满意?
- 满意 → 修复结束
- 需再调整 → 继续描述需求
```

**特殊场景**：
- 素材库无匹配 → 提示用户上传新素材或使用现有素材
- API调用失败 → 提供重试/跳过选项
- 用户描述模糊 → 询问具体调整需求
