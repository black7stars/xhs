# 小红书-图片后处理

针对已生成笔记的图片进行修复、替换、重生成。

## 触发方式

用户针对已生成内容提出问题或指令，自动识别为后处理场景。

**支持场景**：
- 场景A：用户刚执行完 `/小红书` 命令，查看生成结果后提出修改
- 场景B：用户回顾历史笔记，指出某张配图需要调整

## 支持的指令类型

| 用户输入示例 | 识别关键词 | 执行操作 |
|------------|-----------|---------|
| "配图2换一张" | 换/更换/替换 + 配图X | 重新匹配素材 → 生成新图 |
| "重新生成配图3" | 重新生成/重做 + 配图X | 使用原提示词重新调用API |
| "配图1要更多留白" | 配图X + 摆盘调整词 | 修改摆盘说明 → 更新提示词 → 重新生成 |
| "全部重新生成" | 全部/所有 + 重新 | 重新跑第9-11步（配图建议→提示词→生成） |
| "增加2张配图" | 增加/添加 + 数量 | 补充配图规划 → 生成新增配图 |
| "删掉配图4" | 删/去掉 + 配图X | 删除文件并更新笔记 |
| "配图5颜色不对" | 配图X + 颜色/色调问题 | 调整色调参数 → 重新生成 |
| "素材不匹配" | 素材 + 不匹配/不合适 | 重新匹配并询问确认 |

## 工作流程

### 步骤1：识别用户意图

解析用户自然语言，提取：
- **目标配图**：配图X / 全部
- **操作类型**：换素材/重生成/调摆盘/改色调/增删
- **具体要求**：用户描述的调整需求

### 步骤2：定位原文件

基于对话上下文或用户提供的笔记标题，定位：
```
笔记/[笔记标题]/
├── 笔记.md
└── 配图X.jpg
```

### 步骤3：执行修复

**类型A：换素材**
1. 读取原配图建议
2. 重新搜索素材库（排除已用素材）
3. 匹配新素材
4. 生成新配图建议
5. 生成新提示词
6. 调用Banana生成新图
7. 替换原文件
8. 更新笔记.md

**类型B：重新生成**
1. 读取原提示词
2. 调用Banana API
3. 保存新图（覆盖或新建）
4. 更新笔记.md

**类型C：调整摆盘**
1. 读取原配图建议
2. 修改摆盘说明（根据用户要求）
3. 生成新提示词（重点转换新摆盘细节）
4. 调用Banana生成新图
5. 替换原文件
6. 更新笔记.md

**类型D：调整色调/光线**
1. 读取原提示词
2. 修改色彩/光线关键词
   - 暖→冷：移除warm，添加cool tone, blueish
   - 自然光→暖光：natural→warm golden lighting
3. 重新生成
4. 更新文件

**类型E：增加配图**
1. 分析笔记主题
2. 规划新增配图（延续已有风格）
3. 匹配新素材
4. 生成配图建议+提示词
5. 生成图片
6. 追加到笔记.md

**类型F：删除配图**
1. 确认删除配图X
2. 删除配图X.jpg
3. 更新笔记.md（移除对应配图建议和提示词）
4. 重新编号后续配图（可选）

### 步骤4：输出结果

```markdown
✅ 修复完成

📁 修改位置：笔记/[笔记标题]/
🔧 执行操作：[具体操作描述]
⏱️ 耗时：XX秒

**修改后结果**：
- [如果是换素材/调摆盘] 配图X已更新，新素材：[路径]
- [如果是重生成] 配图X已重新生成
- [如果是增删] 配图数量现为X张

📋 更新内容已同步至笔记.md

📌 是否满意？
- 满意 → 修复结束
- 需再调整 → 继续描述需求
```

## 配置文件

**API配置**（复用主命令）：
- Host: https://grsai.dakka.com.cn/v1/draw/nano-banana-pro
- Model: nano-banana-pro

**默认参数**：
- 尺寸：1024x1024
- 步数：30
- CFG：7.5
- 重试次数：3次

## 注意事项

1. **自动识别**：用户输入包含"配图X"或"图片"时自动触发后处理
2. **上下文保持**：修复过程中保持对话上下文，支持连续调整
3. **原子操作**：每次只处理一张配图，避免批量修改出错
4. **备份机制**：覆盖原图前自动备份为配图X_backup.jpg（保留最近3个版本）
5. **同步更新**：所有修改实时同步到笔记.md，确保一致性
