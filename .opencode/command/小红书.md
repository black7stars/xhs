解析用户输入的主题和保存路径，触发完整的13步小红书笔记生成流程。

**参数解析规则**：
- $1: 笔记主题（必填）
- $2, $3, ...: 保存路径（可选，以/结尾则为路径，否则为主题的一部分）

**参数提取逻辑**：
```
# 第一个参数必须是主题
theme = $1

# 检查最后参数是否为路径
last_param = 最后一个非空参数
if last_param 以 "/" 结尾:
    output_path = last_param
    # 主题 = $1 到倒数第二个参数
    theme = 合并 $1 到倒数第二个参数
else:
    output_path = "笔记/[主题]/"
```

**执行流程**：
1. 提取主题和保存路径
2. 调用 agent: xiaohongshu-content 执行完整13步笔记生成流程
3. 传递参数：
   - theme: [提取的主题]
   - output_path: [提取的路径]

**强制要求**：
- agent 必须完成全部13步
- 第9-12步（配图生成）必须实际执行
- 第13步（强制自检）必须调用 skill: content-review 和 prohibited-words
- 只有通过全部自检后方可输出最终结果

**使用示例**：
```
/小红书
春季菜单焕新

/小红书
春季菜单焕新
笔记/2026春季/
```

```

- 配图：配图1.jpg - 配图6.jpg

📊 生成统计：
- 笔记字数：XXX字
- 配图数量：X张

- 失败：X张
- 耗时：MM分SS秒

📋 自检结果：
- 人设检查：✅通过
- 结构检查：✅通过
- 违禁词检查：✅通过

📌 如需调整：
- 换素材 → 使用 `/小红书-图片后处理`
- 重生成 → 说明"配图X重新生成"
- 改摆盘 → 说明"配图X要更多留白"
```

**红线（触发即返工）**：
1. 痛点少于3句或超过5句
2. 观点无具体例子支撑
3. 照搬素材库原菜名
4. 出现"10年"/"蓝带"/"我们"/"一碟盐"
5. 跳过4步结构中的任何一步
