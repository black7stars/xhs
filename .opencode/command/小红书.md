# 小红书笔记生成器

生成符合人设要求的小红书笔记，并自动配图。

## 输入格式

```
/小红书
主题
[保存路径]
```

- **主题**（必填）：笔记主题
- **保存路径**（选填）：以`/`结尾的目录路径，默认保存到`笔记/[日期]/`

**示例**：
```bash
# 默认路径
/小红书
春季菜单焕新

# 自定义路径
/小红书
春季菜单焕新
笔记/2026春季/
```

## 生成流程

### 第1步：确认人设
- 年限：8年
- 背景：ALMA厨艺学院
- 人称："我"（禁用"我们"）
- 语气：掏心窝子聊

参见 `人设.md` 第14-52行

### 第2步：选择创新角度
食材/技法/酱汁/菜单/商业（5选1）

参见 `提示词库/创新角度库.md`

### 第3步：确定内容类型
类型A并列要点/B菜单清单/C经验要点/D框架概述/E详细做法（5选1）

参见 `提示词库/生成规范详解.md` 第六章

### 第4步：撰写正文（4步式）

**第1步：痛点**（3-5句话）
- 充分展开场景
- 4种方式：简述/场景/数据/疑问

**第2步：观点**（3-5句话，6选1）
- 说实话/我发现/记住/其实/说到底/回过头看
- **必须有具体例子支撑**

**第3步：案例**（3选1）
- 详细型/数据型/简述型

**第4步：结尾**（2选1）
- 互动型/夯实观点型

详细示例参见 `提示词库/生成规范详解.md` 第一章

### 第5步：封面标题
- 参考 `提示词库/封面标题库.md`
- 必须含关键词（融合料理培训/Bistro等）
- 标题emoji≤1个
- 总字数≤20字

### 第6步：话题标签
5-10个，参见 `话题库/`

### 第7步：配图规划（4-9张，默认6张）

依据主题+食材匹配，不依赖正文菜品数量。

**配图数量规则**：
- 默认：6张
- 范围：4-9张
- 判定依据：主题丰富度、食材多样性

**配图逻辑**：
- 不区分封面/副图/步骤图，全部是菜品展示图
- **判定批次类型**：
  - 笔记内容为**单道菜品教学/教程** → **Type B（单菜品多视角）**：固定盘子、风格、背景，视角和光线变化展示不同角度
  - 笔记内容为**菜单/清单/多菜品介绍** → **Type A（多菜品批次）**：所有图片保持视角、光线、盘子、风格、背景一致
- 多菜品→各菜品不同角度

**输出**：配图清单（编号1-6或更多）+ 批次配置声明

### 第8步：批次配置锁定

**判定批次类型**：
- 若笔记内容为**单道菜品教学/详细做法** → Type B（单菜品多视角）
- 若笔记内容为**菜单清单/食材对比/多菜品介绍** → Type A（多菜品批次）

**提取锁定配置**（优先级：主题推断 > 素材标签 > 默认）：
1. 根据主题自动推断风格参数（Bistro/高端西餐等）
2. 从素材标签补充盘子、背景等信息
3. 使用默认值兜底

**Type B特殊规则**：
- 锁定：盘子类型、整体风格、背景（整批一致）
- 变化：拍摄视角、光线（按多视角序列变化）

详见 `references/batch-consistency.md`

### 第9-12步：配图生成

调用 `skill: food-image-core` 完成配图全流程：

1. **素材匹配**：基于主题在素材库检索匹配素材（单菜品教学通常匹配1个核心素材）
2. **配图建议**：基于素材YAML和**批次锁定配置**生成摆盘说明
   - Type A：所有图片使用相同视角、光线、盘子、风格、背景
   - Type B：盘子、风格、背景一致，视角和光线按序列变化
3. **提示词生成**：按「图片提示词.md」转换为Banana模型提示词
4. **顺序生图**：调用Banana API生成图片

**参数**：
- 输入类型：text
- 数量：6张（默认）
- 批次类型：根据内容自动判定（Type A或Type B）
- 输出目录：`笔记/[笔记标题]/`

### 第13步：输出结果

**保存结构**：
```

├── 笔记.md          # 正文+配图建议+提示词
```

## ⚠️ 输出前强制自检

**在输出笔记内容之前，必须强制使用以下两个skill完成自检：**

1. **生图质量检查**：调用 `skill: image-validator`
2. **内容质量检查**：调用 `skill: content-review`

详细检查清单参见 `AGENTS.md` 的"强制自检规则"部分。

**只有通过全部检查后，方可输出笔记内容和配图**。

## 红线（触发即返工）

1. 痛点少于3句或超过5句
2. 观点无具体例子支撑
3. 照搬素材库原菜名
4. 出现"10年"/"蓝带"/"我们"/"一碟盐"
5. 跳过4步结构中的任何一步
