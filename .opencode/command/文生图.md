---
description: 基于文字需求生成菜品图片
agent: image-generation
---

解析用户输入，提取数量和需求描述，触发分批图片生成流程（应对 SiliconFlow API 限制）。

**参数解析规则**：
- $1: 图片数量（1-3，可选，默认"2"）⚠️ 受 SiliconFlow API 限制，单次最多3张
- $2, $3, ...: 需求描述（自然语言，支持多行，合并为完整描述）

**参数提取逻辑**：
```
# 尝试解析第一个参数为数量
if $1 是数字 且 1 <= $1 <= 3:
    count = parseInt($1)
    description_start = 2  # 描述从$2开始
else:
    count = 2  # 默认2张（从 .env WORKFLOW_DEFAULT_COUNT 读取）
    description_start = 1  # 描述从$1开始

# 合并所有剩余参数作为需求描述
user_input = 合并参数 $description_start 到末尾
```

**执行流程**：
1. 验证数量参数有效性（1-3，超过则提示并限制为3）
2. 合并所有非数量参数作为完整需求描述
3. 调用 agent: image-generation 执行分批生图流程
4. 传递参数：
   - input_type: "text"
   - count: [提取的数量，已限制为 1-3]
   - user_input: [合并后的需求描述]
   - output_dir: "AI生图/文生图/[当前日期]/"

**分批处理逻辑**：
```
if count > 3:
    提示用户："受 SiliconFlow API 限制，单次最多生成3张图片"
    count = 3

如需生成更多图片，请分多次调用 /文生图 命令
```

**强制要求**：
- agent 必须完成全部7步，不得在第5步停止
- 第6步（生图）和第7步（保存）必须实际执行
- ⚠️ **新增**：单次生成数量不得超过3张（从 .env WORKFLOW_MAX_BATCH_SIZE 读取）
- 输出最终报告（文件位置 + 统计信息）

**使用示例**：
```
/文生图
春季海鲜类，暖色调，俯拍，适合Bistro菜单，要有精致摆盘和留白

/文生图
3
春季海鲜类，暖色调，俯拍，适合Bistro菜单
```

**需求关键词参考**：
- 食材：海鲜类、肉类、蔬菜类、菌菇类、其他
- 色调：暖色调、冷色调、中性色
- 视角：俯拍、45度斜俯拍、平视、特写
- 风格：Bistro轻正餐、高端西餐、融合料理、日式简约
- 摆盘：精致摆盘、极简留白、丰富装饰、对称构图
- 季节：春季、夏季、秋季、冬季
