# 文生图

根据用户文字描述，从素材库匹配并生成菜品图片。

## 输入格式

```
/文生图
[数量，默认4张，范围1-9]
需求描述（支持自然语言）
```

**参数说明**：
- **数量**：可选，默认为4张，范围1-9张
- **需求描述**：食材类型、色调、视角、风格、摆盘要求等

**示例**：
```bash
# 默认生成4张
/文生图
春季海鲜类，暖色调，俯拍，适合Bistro菜单，要有精致摆盘和留白

# 指定生成6张
/文生图
6
春季海鲜类，暖色调，俯拍，适合Bistro菜单
```

## 工作流程

调用 `skill: food-image-core` 完成以下流程：

### 第1步：需求解析
从用户输入提取结构化需求：
- **食材类型**：海鲜类/肉类/蔬菜类/菌菇类/其他
- **色调偏好**：暖色调/冷色调/中性色
- **视角要求**：俯拍/45度斜俯拍/平视/特写
- **风格定位**：Bistro轻正餐/高端西餐/融合料理
- **摆盘要素**：留白比例、装饰复杂度、盘子类型
- **场景/季节**：春季/夏季/秋季/冬季

### 第2步：素材匹配
在 `素材库/图片素材库/` 中检索匹配素材：
- 按匹配度排序（食材40% + 色调25% + 季节20% + 风格15%）
- 返回前N个最佳匹配素材（N=指定生成数量）

### 第3步：批次配置锁定

在生成配图建议前，确定批次类型并锁定配置：

**判定批次类型**：
- 若匹配素材数 > 1 → Type A（多菜品批次）
- 若匹配素材数 = 1 且 生成数量 > 1 → Type B（单菜品多视角）

**提取锁定配置**（优先级：用户输入 > 主题推断 > 素材标签 > 默认）：
1. 从用户自然语言输入提取显式指定的视角/光线/盘子/风格/背景
2. 根据主题自动推断未指定的参数
3. 从首个素材YAML标签补充
4. 使用默认值兜底

**配置保存**：
- 拍摄视角：[俯拍/45度/平视/特写]
- 光线：[自然光/暖光/冷光]
- 盘子：[陶瓷白盘/粗陶盘/石板/...]
- 整体风格：[Bistro轻正餐/高端西餐/...]
- 背景：[木质桌面/大理石/纯色背景/...]

详见 `references/batch-consistency.md`

### 第4步：配图建议生成
基于素材YAML标签和**批次锁定配置**，按「提示词库/配图规范详解.md」生成详细配图建议：

- **Type A批次**：所有配图建议使用相同的视角、光线、盘子、风格、背景
- **Type B批次**：所有配图建议使用相同的盘子、风格、背景，视角和光线按多视角规范变化

### 第5步：提示词生成
按「提示词库/图片提示词.md」将配图建议转换为Banana模型提示词。

### 第6步：顺序生图
调用Banana API顺序生成图片：
- 比例：3:4（小红书推荐）
- 尺寸：2K
- 重试机制：失败3次后跳过，保存简化提示词

```

├── 配图1.jpg
├── 配图2.jpg
├── ...
└── 生成记录.md
```

**生成记录.md 内容**：
```markdown

## 生成时间
2026-02-28 14:32:15

## 用户输入

## 批次配置

- 锁定配置：
  - 拍摄视角：45度斜俯拍
  - 光线：自然侧光
  - 盘子：粗陶浅盘
  - 整体风格：Bistro轻正餐

1. 素材库/图片素材库/海鲜类/绯焰金鲷_001.yaml (匹配度92%)
2. ...

[提示词列表]

## 生成结果
- 成功：X张
- 失败：Y张
- 耗时：MM分SS秒
```
