---
description: 基于文字需求生成菜品图片
agent: image-generation
---

解析用户输入，提取数量和需求描述，触发完整的7步图片生成流程。

**参数解析规则**：
- $1: 图片数量（1-9，可选，默认"4"）
- $2, $3, ...: 需求描述（自然语言，支持多行，合并为完整描述）

**参数提取逻辑**：
```
# 尝试解析第一个参数为数量
if $1 是数字 且 1 <= $1 <= 9:
    count = parseInt($1)
    description_start = 2  # 描述从$2开始
else:
    count = 4  # 默认4张
    description_start = 1  # 描述从$1开始

# 合并所有剩余参数作为需求描述
user_input = 合并参数 $description_start 到末尾
```

**执行流程**：
1. 验证数量参数有效性，无效则使用默认值4
2. 合并所有非数量参数作为完整需求描述
3. 调用 agent: image-generation 执行完整7步生图流程
4. 传递参数：
   - input_type: "text"
   - count: [提取的数量]
   - user_input: [合并后的需求描述]
   - output_dir: "AI生图/文生图/[当前日期]/"

**强制要求**：
- agent 必须完成全部7步，不得在第5步停止
- 第6步（生图）和第7步（保存）必须实际执行
- 输出最终报告（文件位置 + 统计信息）

**使用示例**：
```
/文生图
春季海鲜类，暖色调，俯拍，适合Bistro菜单，要有精致摆盘和留白

/文生图
6
春季海鲜类，暖色调，俯拍，适合Bistro菜单
```

**输出结果**：
```
✅ 生成完成

📁 文件位置：AI生图/文生图/[日期]/
📊 生成统计：
- 总计：X张
- 成功：X张
- 失败：X张
- 耗时：MM分SS秒

📄 文件列表：
✅ 配图1.jpg
✅ 配图2.jpg
...

📝 生成记录已保存至：AI生图/文生图/[日期]/生成记录.md
```

**需求关键词参考**：
- 食材：海鲜类、肉类、蔬菜类、菌菇类、其他
- 色调：暖色调、冷色调、中性色
- 视角：俯拍、45度斜俯拍、平视、特写
- 风格：Bistro轻正餐、高端西餐、融合料理、日式简约
- 摆盘：精致摆盘、极简留白、丰富装饰、对称构图
- 季节：春季、夏季、秋季、冬季
