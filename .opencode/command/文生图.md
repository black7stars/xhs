# 文生图

根据用户文字描述，从素材库匹配并生成菜品图片。

## 输入格式

```
/文生图
[数量，默认4张，范围1-9]
需求描述（支持自然语言）
```

**参数说明**：
- **数量**：可选，默认为4张，范围1-9张
- **需求描述**：食材类型、色调、视角、风格、摆盘要求等

**示例**：
```bash
# 默认生成4张
/文生图
春季海鲜类，暖色调，俯拍，适合Bistro菜单，要有精致摆盘和留白

# 指定生成6张
/文生图
6
春季海鲜类，暖色调，俯拍，适合Bistro菜单
```

## 工作流程

调用 `skill: food-image-core` 完成以下流程：

### 第1步：需求解析
从用户输入提取结构化需求：
- **食材类型**：海鲜类/肉类/蔬菜类/菌菇类/其他
- **色调偏好**：暖色调/冷色调/中性色
- **视角要求**：俯拍/45度斜俯拍/平视/特写
- **风格定位**：Bistro轻正餐/高端西餐/融合料理
- **摆盘要素**：留白比例、装饰复杂度、盘子类型
- **场景/季节**：春季/夏季/秋季/冬季

### 第2步：素材匹配
在 `素材库/图片素材库/` 中检索匹配素材：
- 按匹配度排序（食材40% + 色调25% + 季节20% + 风格15%）
- 返回前N个最佳匹配素材（N=指定生成数量）

### 第3步：配图建议生成
基于素材YAML标签，按「提示词库/配图规范详解.md」生成详细配图建议，包含6项摆盘说明。

### 第4步：提示词生成
按「提示词库/图片提示词.md」将配图建议转换为Banana模型提示词。

### 第5步：顺序生图
调用Banana API顺序生成图片：
- 尺寸：1242×1660（小红书3:4推荐尺寸）
- 步数：30，CFG：7.5
- 重试机制：失败3次后跳过，保存简化提示词

## 需求关键词参考

| 维度 | 可选值 |
|-----|-------|
| 食材 | 海鲜类、肉类、蔬菜类、菌菇类、其他 |
| 色调 | 暖色调(金黄橙红)、冷色调(蓝绿青)、中性色 |
| 视角 | 俯拍(90度)、45度斜俯拍、平视、特写 |
| 风格 | Bistro轻正餐、高端西餐、融合料理、日式简约 |
| 摆盘 | 精致摆盘、极简留白、丰富装饰、对称构图 |
| 季节 | 春季、夏季、秋季、冬季 |
