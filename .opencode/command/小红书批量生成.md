description: 批量生成小红书餐饮融合料理笔记
agent: xiaohongshu-content
---

解析多个主题，循环调用小红书笔记生成流程，自动去重。

**参数解析规则**：
- $1, $2, ..., $N-1: 多个主题（至少1个）
- $N（最后一个）: 保存路径（可选，以/结尾则为路径）

**参数提取逻辑**：
```
# 收集所有参数
themes = []
output_path = null

for 每个参数:
    if 参数以 "/" 结尾 且 是最后一个参数:
        output_path = 参数
    else:
        themes.append(参数)

if themes 为空:
    提示用户至少提供一个主题
    return

if output_path 为 null:
    output_path = "笔记/批量生成-[日期]/"
```

**执行流程**：
1. 提取所有主题和保存路径
2. 初始化批次状态追踪器
3. 循环每个主题：
   a. 调用 agent: xiaohongshu-content
   b. 传递参数：
      - theme: [当前主题]
      - output_path: [基础路径/主题名称/]
      - batch_state: [批次状态，用于去重]
   c. 接收返回的已用元素，更新批次状态
   d. 记录成功/失败结果
4. 生成批量报告

**状态追踪**（用于去重）：
- 开场白变体
- 创新角度
- 素材组合
- 观点方式
- 菜品名称

**去重规则**：
- 角度：与本次已用不重复
- 组合：与本次已用不重复
- 观点方式：与上1篇不重复（至少隔2篇）
- 菜名：与本次已用不重复

**强制要求**：
- 每篇必须通过自检，失败则跳过继续下一篇
- 单次建议≤7个主题
- 失败的笔记可单独用 `/小红书` 命令重试

**使用示例**：
```
/小红书批量生成
春季菜单焕新
2026融合料理趋势

/小红书批量生成
春季菜单焕新
2026融合料理趋势
笔记/2026春季/
```
