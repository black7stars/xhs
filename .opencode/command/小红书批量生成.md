# 小红书批量生成器

批量生成多篇笔记，自动去重。

## 输入格式

```
/小红书批量生成
主题1
主题2
...
[保存路径]
```

**路径识别**：最后一行以`/`结尾为保存路径，否则使用默认路径。

**示例**：
```bash
# 默认路径
/小红书批量生成
春季菜单焕新
2026融合料理趋势

# 自定义路径
/小红书批量生成
春季菜单焕新
2026融合料理趋势
笔记/2026春季/
```

## 批量特有逻辑

### 1. 状态追踪
记录批次内已用元素：
- 开场白变体
- 创新角度
- 素材组合
- 观点方式
- 菜品名称

### 2. 去重规则
- 角度：与本次已用不重复
- 组合：与本次已用不重复
- 观点方式：与上1篇不重复（至少隔2篇）
- 菜名：与本次已用不重复

### 3. 单篇生成
每篇主题调用 `/小红书` 命令逻辑：
1. 选择未使用的变体/角度/组合
2. 生成正文（4步结构）
3. 执行强制自检
4. 输出并记录

详细生成规则参见 `/小红书` 命令和 `提示词库/生成规范详解.md`

## 错误处理

- **成功**：记录文件路径，更新状态追踪
- **失败**：记录问题，生成解决建议，**跳过继续下一篇**

## 批量报告格式

```
📊 批量生成报告

总计：X 个主题
✅ 成功：Y 个
❌ 失败：Z 个

━━━━━━━━━━━━━━━━━━━━━━━

✅ 成功项目：
1. [主题]
   文件：[路径]
   角度：[使用的角度]
   组合：[使用的组合]

❌ 失败项目：
1. [主题]
   问题：[原因]
   建议：[解决方法]

━━━━━━━━━━━━━━━━━━━━━━━

📋 状态追踪汇总：
已用变体：[列表]
已用角度：[列表]
已用组合：[列表]
已用菜名：[列表]
```

## 注意事项

1. 单次建议≤7个主题
2. 每篇必须通过自检
3. 失败的笔记可单独用 `/小红书` 命令重试
4. 详细规范参见 `提示词库/生成规范详解.md`
