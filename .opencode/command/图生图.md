# 图生图

分析用户上传图片，自动入库后生成风格变体图。

## 输入格式

支持两种形式：

**形式A：仅上传图片（自动分析）**
```
/图生图
[数量，默认4张]
```
+ 上传图片附件

**形式B：上传图片+文字需求**
```
/图生图
6
想要冷色调版本，更多留白，适合冬季菜单
```
+ 上传图片附件

## 工作流程

调用 `skill: food-image-core` 完成以下流程：

### 第1步：图片分析
分析上传图片并生成YAML标签：
- 识别菜品内容、食材分类
- 分析摆盘结构、色彩、光线
- **自动保存**至素材库对应分类目录

### 第2步：需求融合（如用户提供）
将用户文字需求与图片分析结果融合：
- 保留原图核心元素（主体、摆盘结构）
- 叠加用户指定调整（色调/构图/风格等）

### 第3步：批次配置锁定

基于分析结果和需求，锁定批次配置：

**判定批次类型**：
- 若生成数量 = 1 → 单图生成
- 若生成数量 > 1 → Type B（单菜品多视角，基于同一图片生成变体）

**提取锁定配置**（优先级：用户输入 > 图片分析 > 默认）：
1. 从用户自然语言输入提取显式指定的参数
2. 从图片分析结果提取（继承原图的盘子、风格、背景）
3. 使用默认值兜底

**配置保存**：视角和光线按Type B规则变化，其余参数锁定。

详见 `references/batch-consistency.md`

### 第4步：配图建议生成
基于图片分析结果和**批次锁定配置**，按「提示词库/配图规范详解.md」生成配图建议。

**Type B规则**：
- 锁定：盘子、整体风格、背景（继承自原图或用户指定）
- 变化：视角和光线按多视角序列变化

### 第5步：提示词生成
按「提示词库/图片提示词.md」将配图建议转换为Banana模型提示词。

### 第6步：顺序生图
调用Banana API顺序生成图片。

### 第7步：文件保存

**输出结构**：
```
AI生图/图生图/[日期]/
├── 配图1.jpg          # 生成的变体图
├── 配图2.jpg
├── ...
├── 原始图片_001.jpg    # 用户上传图备份
└── 生成记录.md        # 含分析结果、批次配置和用户需求
```

**生成记录.md 内容**：
```markdown

## 用户输入
[原始需求文字]

## 批次配置

  - 整体风格：Bistro轻正餐（继承原图）
  - 背景：复古木质桌面（继承原图）
- 变化序列：
  - 配图1：俯拍 + 自然光
  - 配图2：45度斜俯拍 + 暖光
  - 配图3：平视 + 侧光
  - 配图4：特写 + 顶光
```

## 支持的变体调整

用户可在需求中指定：
- **色调**：暖色→冷色、冷色→暖色
- **留白**：增加/减少留白比例
- **季节**：春季→冬季等季节氛围调整
- **光线**：自然光→暖光/冷光
- **风格**：Bistro→高端西餐等风格转换
- **批次类型强制**：用户可指定"多角度"触发Type B，或"统一风格"触发Type A
