---
description: 基于上传图片生成变体菜品图
agent: image-generation
---

接收用户上传图片，解析数量和调整需求，触发完整的7步图片生成流程（基于图片分析+变体生成）。

**参数解析规则**：
- $1: 图片数量（1-9，可选，默认"4"）
- $2, $3, ...: 调整需求（可选，如"冷色调版本，更多留白"）
- 图片附件：必须上传的图片文件

**参数提取逻辑**：
```
# 检查是否有图片附件
if 无图片附件:
    提示用户上传图片
    return

# 尝试解析第一个参数为数量
if $1 是数字 且 1 <= $1 <= 9:
    count = parseInt($1)
    adjustment_start = 2
else:
    count = 4  # 默认4张
    adjustment_start = 1

# 合并剩余参数作为调整需求
adjustment_desc = 合并参数 $adjustment_start 到末尾
```

**执行流程**：
1. 检查图片附件，如未上传则提示用户
2. 提取数量参数（默认4）
3. 合并剩余参数作为调整需求描述
4. 调用 agent: image-generation 执行完整7步生图流程
5. 传递参数：
   - input_type: "image"
   - count: [提取的数量]
   - uploaded_image: [图片附件]
   - adjustment_desc: [调整需求描述]
   - output_dir: "AI生图/图生图/[当前日期]/"

**强制要求**：
- agent 必须完成全部7步，不得在第5步停止
- 第1步包含图片分析和自动入库
- 第6步（生图）和第7步（保存）必须实际执行
- 输出最终报告（文件位置 + 统计信息）

**使用示例**：
```
/图生图
[上传图片附件]

/图生图
6
做成冷色调，适合夏季菜单，增加留白
[上传图片附件]
```

**支持的变体调整**：
- 色调：暖色→冷色、冷色→暖色
- 留白：增加/减少留白比例
- 季节：春季→冬季等季节氛围调整
- 光线：自然光→暖光/冷光
- 风格：Bistro→高端西餐等风格转换
