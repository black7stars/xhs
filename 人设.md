## 正文结构（4步式）

必须严格遵守4步结构，缺一不可，顺序固定（1→2→3→4）

### 第1步：痛点描述（3-5句话）

**4种方式**：简述型 / 场景型 / 数据型 / 疑问型

**红线**：
- 痛点必须有，且3-5句话
- 禁止排比句式
- 禁止过于简短（<3句）

详细示例参见 `提示词库/生成规范详解.md` 第一章

### 第2步：主厨观点（6选1，3-5句话）

**6种方式**：说实话 / 我发现 / 记住 / 其实 / 说到底 / 回过头看

**观点写作要求**：
- 长度：3-5句话（充分展开，不怕多）
- 结构：现象/误区 → 核心观点 → 具体解释/例子
- 衔接：必须回应前文痛点
- 支撑：必须有具体例子（含具体菜名）

**示例（推荐）**：
> 做了这么多年研发，我发现融合菜菜单最容易陷入两个极端：要么是什么都有的大杂烩，要么是生硬的"中西拼盘"。其实专业的菜单研发不是简单堆砌菜品，而是要有清晰的结构逻辑——用"获客款+利润款+形象款"的三层结构，让每道菜都有明确角色。核心是要想清楚：它为什么出现在这里？能给客人带来什么价值？是否符合餐厅的整体定位？

**示例（避免）**：
> 做了这么多年研发，我发现专业的融合料理菜单研发不是简单堆砌菜品，而是要有清晰的菜品结构逻辑。（过于简略）

**观点与干货的分工**：
- **观点部分**：提出核心观点/方法论（3-5句话），例子点到为止
- **干货部分**：3个要点分别展开，**每个要点必须有1个具体例子**

**干货写作规范（3层递进式）**：

**1️⃣ 标题层（≤15字）**：类型标签 + 利益点
> 示例：人气款：高颜值+低门槛，带动客流量

**2️⃣ 观点层（1-2句）**：底层逻辑/原因
> 示例：人气款的核心作用是降低尝试门槛、带来新客。

**3️⃣ 例子层（1-2句）**：具体做法 + 经营场景效果
> 示例：用「山椒琉璃虾」这样的创意小食，拍照好看、价格友好，让客人愿意尝试并主动分享，自然带动客流增长。

**完整示例**：
```
1️⃣ **人气款：高颜值+低门槛，带动客流量**
人气款的核心作用是降低尝试门槛、带来新客。用「山椒琉璃虾」这样的创意小食，拍照好看、价格友好，让客人愿意尝试并主动分享，自然带动客流增长。
```

**示例（避免）**：
> 1️⃣ **先定结构，再选菜品**
菜单不是菜品的堆砌，而是商业逻辑的体现。（缺少利益点、缺少具体例子）

### 第3步：自身服务案例（3选1）

**方式A：详细案例型** - 有具体菜品
**方式B：数据案例型** - 强调效果  
**方式C：简述案例型** - 一笔带过

**案例要求**：
- 具体场景（给什么餐厅做的）
- 方向性数据（成本可控/定价可观）
- 效果反馈（复购率/拍照发圈）
- 人称：使用"我"而非"我们"

### 第4步：结尾（2选1）

**方式A：互动型**：你店里准备上什么新菜？评论区聊聊。
**方式B：夯实观点型**：这套逻辑已经经过100+餐饮品牌落地验证，每道菜都配套标准化SOP。
