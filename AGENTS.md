## 核心规则

### 基础要求
- **标题**：≤20字，1个emoji以内，含关键词
- **正文**：300-500字
- **标签**：5-10个
- **结构**：4步式（痛点→观点→案例→结尾）
- **人称**：第一人称"我"

### 人设（3要素）
- 年限：8年
- 背景：ALMA厨艺学院
- 人称："我"（禁用"我们"）

### 4步结构
1. **痛点**：3-5句话，充分展开
2. **观点**：3-5句话，**必须有具体例子**
3. **案例**：具体场景+数据+效果
4. **结尾**：互动型/夯实观点型

详细说明参见 `提示词库/生成规范详解.md`

## 红线（触发即返工）

1. 痛点少于3句或超过5句
2. 观点无具体例子支撑
3. 照搬素材库原菜名
4. 出现"10年"/"蓝带"/"我们"/"一碟盐"
5. 跳过4步结构中的任何一步
6. 连续2篇使用相同观点方式
7. 与上一篇结构相似度>50%

## 创作流程

1. **确认人设**（参见 `人设.md`）
2. **选择角度**（5大角度，参见创新角度库）
3. **确定类型**（5种类型，参见生成规范详解）
4. **撰写正文**（严格4步结构）
5. **强制自检**（逐项确认，输出前展示结果）
6. **违禁词检测**（使用prohibited-words skill）
7. **输出文件**

## Review检查

**必须执行**：生成后使用 `skill: content-review` 检查

**通过标准**：
- 人设一致（8年+ALMA+我）
- 结构完整（4步齐全）
- 痛点3-5句
- 观点有例子
- 无违禁词

详细检查项参见 `content-review/SKILL.md`

## 素材引用

**必须**：
- 重新命名菜品（避免侵权）
- 混合不同素材表达方式
- 改编而非复制

**禁止**：
- 照搬素材库
- 连续使用相同结构
- 使用文案公式一字不差

## 技术操作规范（防错误指南）

### 避免 EISDIR 错误（目录写入错误）

**问题描述**：
当使用 `write` 或 `edit` 工具时，如果目标路径是**目录**而非文件，会报错：
```
Error: EISDIR: illegal operation on a directory, open '/path/to/dir'
```

**根本原因**：
- `write` 工具：会**覆盖**目标路径的内容
- `edit` 工具：需要读取文件后进行**修改**
- 这两个工具都**不能操作目录**，只能操作文件

**预防措施**：

1. **写入前必须确认路径类型**
   ```bash
   # 使用bash检查路径是文件还是目录
   ls -la /path/to/target
   
   # 或使用test命令
   [ -f /path/to/target ] && echo "是文件" || echo "不是文件"
   [ -d /path/to/target ] && echo "是目录" || echo "不是目录"
   ```

2. **使用glob工具查找文件，避免手动拼接路径**
   - ✅ 正确：使用 `glob` 搜索具体文件
   - ❌ 错误：手动拼接路径字符串写入

3. **write工具使用规范**
   - ✅ 正确：`filePath` 必须是**具体文件路径**，以 `.md`、`.txt` 等扩展名结尾
   - ❌ 错误：`filePath` 以 `/` 结尾或指向目录
   - 示例：
     - ✅ `/Users/xxx/project/README.md`
     - ❌ `/Users/xxx/project/` 或 `/Users/xxx/project`

4. **edit工具使用规范**
   - **必须**先用 `read` 工具读取文件内容
   - 确认文件存在且是文件类型
   - 编辑时使用准确的 `oldString`（包含上下文）

**错误恢复**：
如果不慎遇到 EISDIR 错误：
1. 立即停止当前操作
2. 使用 `bash` 检查目标路径类型
3. 确认是文件后重新执行
4. 如果是目录，需要重新确定正确的文件路径

**最佳实践**：
- 写操作前，先用 `bash` 命令验证路径
- 不确定时，先用 `glob` 搜索确认文件存在
- 遵循"先读再写"原则（read → edit）
- 使用绝对路径而非相对路径
