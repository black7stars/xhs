# AI生图目录

本目录用于存储AI生成的菜品图片，分为文生图和图生图两个子目录。

## 目录结构

```
AI生图/
├── 文生图/               # 文字生成的图片
│   ├── .gitkeep
│   └── [日期]/
│       ├── 配图1.jpg
│       ├── 配图2.jpg
│       └── 生成记录.md
└── 图生图/               # 基于图片生成的变体
    ├── .gitkeep
    └── [日期]/
        ├── 配图1.jpg
        ├── 配图2.jpg
        ├── 原始图片_001.jpg
        └── 生成记录.md
```

## 文生图

**功能**：根据文字描述生成菜品图片

**使用命令**：
```bash
/文生图
[数量，默认4张]
需求描述
```

**保存位置**：`AI生图/文生图/[日期]/`

**生成记录**：
每个批次自动生成 `生成记录.md`，包含：
- 生成时间
- 用户输入
- 匹配素材列表
- 生成的提示词
- 生成结果统计

## 图生图

**功能**：基于上传图片生成风格变体

**使用命令**：
```bash
/图生图
[数量，默认4张]
[调整需求描述]
+ 上传图片
```

**保存位置**：`AI生图/图生图/[日期]/`

**文件说明**：
- `配图X.jpg` - 生成的变体图
- `原始图片_001.jpg` - 用户上传的原图备份
- `生成记录.md` - 生成记录和分析结果

## 图片规格

- **尺寸**：1242×1660（小红书3:4推荐）
- **格式**：JPG
- **质量**：高清
- **风格**：Bistro轻正餐、高端西餐、融合料理

## 使用流程

### 文生图流程
1. 用户提供文字需求
2. 系统匹配素材库
3. 生成配图建议
4. 转换为Banana提示词
5. 调用API生成图片
6. 保存到本目录

### 图生图流程
1. 用户上传图片
2. 系统分析图片并入库
3. 融合用户调整需求
4. 生成变体提示词
5. 调用API生成图片
6. 保存到本目录

## 注意事项

- 本目录已通过 `.gitignore` 配置，不会提交到 git
- 仅保留 `.gitkeep` 文件以保持目录结构
- 生成的图片保存在本地，请自行备份重要内容
- 建议定期清理旧图片，释放磁盘空间

## 后处理

如需调整已生成的图片：
- 使用 `/小红书-图片后处理` 命令
- 支持：换素材、重生成、调摆盘、改色调、增删图片
