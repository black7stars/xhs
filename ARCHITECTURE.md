# 小红书运营系统架构文档

## 三层架构总览

```
┌─────────────────────────────────────────────────────────────┐
│  Command 层 (.opencode/command/*.md)                        │
│  职责：用户入口、参数解析、路由到 Agent                      │
│  ├── 文生图.md → agent: image-generation                    │
│  ├── 图生图.md → agent: image-generation                    │
│  ├── 小红书.md → agent: xiaohongshu-content                 │
│  ├── 小红书批量生成.md → 循环调用小红书 agent               │
│  └── ...                                                    │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│  Agent 层 (.opencode/agents/*.md)                           │
│  职责：工作流控制器，执行完整流程                            │
│  ├── image-generation: 7步图片生成流程                       │
│  │   ├── 第1-5步：准备阶段（调用 Skill）                     │
│  │   ├── 第5.5步：后验检查（调用 image-validator）           │
│  │   └── 第6-7步：执行阶段（直接调用API/文件操作）           │
│  └── xiaohongshu-content: 13步笔记生成流程                   │
│      ├── 第1-8步：内容准备                                   │
│      ├── 第9-12步：图片生成（调用image-generation agent）    │
│      └── 第13步：自检（调用 Skill）                          │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│  Skill 层 (.opencode/skills/*/SKILL.md)                     │
│  职责：原子能力单元，供 Agent 按需调用                       │
│  ├── food-image-core: 需求解析、素材匹配、配图建议、提示词   │
│  ├── image-validator: 生图质量后验检查（第5步后调用）        │
│  ├── prohibited-words: 违禁词检测                           │
│  └── content-review: 内容质量检查                           │
└─────────────────────────────────────────────────────────────┘
```

## 职责边界

| 层级 | 核心职责 | 禁止行为 |
|-----|---------|---------|
| **Command** | 接收输入、解析参数、路由到 Agent | 不执行具体逻辑、不调用 Skill |
| **Agent** | 控制工作流、决定何时调用 Skill、整合结果 | 不重复 Skill 的原子能力逻辑 |
| **Skill** | 提供原子能力、完成具体任务、返回结构化结果 | 不控制流程、不决定何时被调用 |

## 输入格式标准

所有 Command 统一使用换行输入格式：

```
/命令名
参数1
参数2
参数3...
```

示例：
```
/文生图
6
春季海鲜类，暖色调，俯拍，适合Bistro菜单

/小红书
春季菜单焕新
笔记/2026春季/
```

## 协作流程

### 文生图流程

1. 用户输入：`/文生图` + 换行参数
2. Command 解析参数
3. Command 调用 `agent: image-generation`
4. Agent 执行：
   - 第1-5步：调用 `skill: food-image-core` 各模块
   - 第5.5步：调用 `skill: image-validator` 后验检查
   - 第6步：调用 Banana API 生图
   - 第7步：保存图片 + 生成记录.md（唯一文档）
5. Agent 输出最终报告

### 小红书流程

1. 用户输入：`/小红书` + 换行参数
2. Command 解析参数后调用 `agent: xiaohongshu-content`
3. Agent 执行13步：
   - 第1-8步：生成笔记内容
   - 第9-12步：内部调用 `agent: image-generation`
   - 第13步：调用 `skill: image-validator`、`skill: content-review` 和 `skill: prohibited-words`
4. Agent 输出最终报告

## 批次一致性规则

### Type A（多菜品批次）

**必须完全相同**（所有配图）：
- ✅ 盘子选择（材质/形状/尺寸/颜色）
- ✅ 整体风格（Bistro/高端西餐等）
- ✅ 背景（木质/大理石等）
- ✅ 拍摄视角（俯拍/45度等）
- ✅ 光线（自然光/暖光等）

### Type B（单菜品多视角）

**锁定相同**（所有配图）：
- ✅ 盘子选择
- ✅ 整体风格
- ✅ 背景

**按序列变化**：
- 📷 视角序列：俯拍 → 45度 → 平视 → 特写
- 💡 光线序列：自然光 → 暖光 → 侧光 → 顶光

### image-validator 检查

新增第5步：批次一致性检查
- Type A：检查所有锁定字段是否完全相同
- Type B：检查锁定字段相同 + 变化字段按序列
