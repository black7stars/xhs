# 图片素材库使用指南

## 目录结构

```
素材库/图片素材库/
├── 海鲜类/          # 海鲜类菜品图片
├── 肉类/            # 肉类菜品图片
├── 蔬菜类/          # 蔬菜类菜品图片
├── 菌菇类/          # 菌菇类菜品图片
└── 其他/            # 其他分类（甜点、饮品等）
```

## 使用流程

### 1. 上传图片

将菜品图片上传到对应分类目录：
- 海鲜 → `海鲜类/`
- 牛肉/猪肉/羊肉 → `肉类/`
- 蔬菜/沙拉 → `蔬菜类/`
- 菌菇 → `菌菇类/`

**命名规范**：
```
[菜品描述]_[编号].jpg

示例：
- 蟹肉塔_001.jpg
- 黑猪肉_003.jpg
- 三文鱼_002.jpg
```

### 2. 创建YAML标签文件

为每张图片创建同名YAML描述文件（与图片同名，扩展名改为.yaml）：

**示例**：`蟹肉塔_001.yaml`

```yaml
filename: 蟹肉塔_001.jpg
path: 素材库/图片素材库/海鲜类/

# 基础信息
primary_ingredient: 海鲜类          # 一级分类：海鲜/肉类/蔬菜/菌菇
dish_type: 前菜                     # 前菜/主菜/甜点/饮品
cuisine_style: 融合料理              # 中式/西式/融合/日料

# 视觉特征
main_colors:
  - 红色    # 主色调
  - 白色    # 辅色调
  - 绿色    # 点缀色
  - 黄色
composition: 俯拍                    # 俯拍/平视/特写/对角线
presentation_style: 左侧留白              # 左侧留白/右侧留白/中心构图/不规则分布
tableware: 纯白哑光圆形盘            # 餐具类型
tableware_size: 25-28cm             # 餐具尺寸

# 场景标签
season: 春季                        # 春季/夏季/秋季/冬季/全年
occasion: 日常                      # 节日/日常/商务/约会
lighting: 自然光                     # 自然光/暖光/冷光/混合光

# 内容描述（用于AI匹配）
content_description: >
  白色极简盘中5个红色水滴形甜菜根蟹肉塔，
  甜菜根天然红色外皮呈叶片状造型，内馅蟹肉沙拉，
  顶部黄色小花碎密集点缀，绿色旱金莲叶每塔2-3片，
  呈不规则分布在右上2个+右下3个，左侧1/3大面积留白

# 摆盘细节（关键）
plating_details:
  plate: 纯白哑光圆形盘，直径25-28cm，无边沿设计
  logic: 黄金分割构图，5个塔壳呈"右上聚集+右下延伸"不规则群组，形成从右上向左下对角线视觉流动
  spacing: 塔壳间距2-3cm，保持独立性又形成群组感
  decoration: >
    旱金莲叶每塔2-3片，叶片方向错落有致，黄色碎花密集均匀覆盖
  negative_space: 左侧1/3大面积留白
  overall_style: 法式精致感fine dining，符合Bistro定位
```

### 3. YAML字段说明

| 字段 | 必填 | 说明 | 示例值 |
|-----|------|------|--------|
| filename | 是 | 图片文件名 | 蟹肉塔_001.jpg |
| path | 是 | 文件路径 | 素材库/图片素材库/海鲜类/ |
| primary_ingredient | 是 | 一级分类 | 海鲜类/肉类/蔬菜类/菌菇类 |
| dish_type | 是 | 菜品类型 | 前菜/主菜/甜点/饮品 |
| cuisine_style | 否 | 菜系风格 | 中式/西式/融合/日料 |
| main_colors | 是 | 主色调列表 | [红色, 白色, 绿色] |
| composition | 是 | 拍摄视角 | 俯拍/平视/特写 |
| plating_style | 是 | 摆盘留白 | 左侧留白/右侧留白 |
| tableware | 是 | 餐具类型 | 纯白哑光圆形盘 |
| tableware_size | 否 | 餐具尺寸 | 25-28cm |
| season | 是 | 适用季节 | 春季/夏季/秋季/冬季/全年 |
| occasion | 否 | 适用场景 | 日常/节日/商务 |
| lighting | 是 | 光线类型 | 自然光/暖光/冷光 |
| content_description | 是 | 详细描述 | 多行文本，描述画面内容 |
| plating_details | 是 | 摆盘细节 | 包含6个子项（见下方） |

**plating_details子项**：

| 子项 | 说明 |
|-----|------|
| plate | 盘子材质+形状+尺寸+边沿 |
| logic | 构图法则+视觉流动方向 |
| spacing | 间距+分布特点 |
| decoration | 装饰物+分布方式+效果 |
| negative_space | 留白位置+比例 |
| overall_style | 风格定位+场景契合 |

### 4. 快速创建YAML模板

使用以下模板快速填写：

```yaml
filename: [图片名].jpg
path: 素材库/图片素材库/[分类]/

primary_ingredient: [海鲜类/肉类/蔬菜类/菌菇类]
dish_type: [前菜/主菜/甜点/饮品]
cuisine_style: [中式/西式/融合/日料]

main_colors:
  - [主色]
  - [辅色]
  - [点缀色]

composition: [俯拍/平视/特写/对角线]
presentation_style: [左侧留白/右侧留白/中心构图/不规则分布]
tableware: [餐具描述]
tableware_size: [尺寸]

season: [春季/夏季/秋季/冬季/全年]
occasion: [日常/节日/商务/约会]
lighting: [自然光/暖光/冷光/混合光]

content_description: >
  [详细描述图片内容]

plating_details:
  plate: [盘子描述]
  logic: [摆盘逻辑]
  spacing: [间距控制]
  decoration: >
    [装饰法则]
  negative_space: [留白描述]
  overall_style: [整体风格]
```

### 5. 批量上传

如需批量上传多张图片：

1. 按分类放入对应目录
2. 批量创建YAML文件（可使用脚本）
3. 确保每张图片都有同名yaml文件

**示例批量结构**：
```
海鲜类/
├── 蟹肉塔_001.jpg
├── 蟹肉塔_001.yaml
├── 三文鱼_001.jpg
├── 三文鱼_001.yaml
├── 龙虾_002.jpg
└── 龙虾_002.yaml
```

## 匹配逻辑

系统根据以下优先级匹配图片：

1. **主题匹配**（权重40%）：主题关键词匹配
2. **食材匹配**（权重30%）：primary_ingredient匹配
3. **色调匹配**（权重15%）：main_colors匹配
4. **摆盘风格**（权重15%）：plating_style匹配

## 注意事项

- 每张图片必须有同名yaml文件，否则无法被系统识别
- yaml文件名必须与图片完全一致（仅扩展名不同）
- content_description和plating_details越详细，AI匹配效果越好
- 建议每个分类下保持10-30张图片，便于多样选择
- 定期更新素材库，添加时令菜品图片
