# 素材库

本目录存储所有参考素材，包括图片素材、菜品描述和笔记文本素材。

```
素材库/
├── 图片素材库/          # 菜品图片及YAML标签（★核心资产）
│   ├── 海鲜类/          # 海鲜类菜品
│   ├── 肉类/            # 肉类菜品
│   ├── 蔬菜类/          # 蔬菜类菜品
│   ├── 菌菇类/          # 菌菇类菜品
│   └── 其他/            # 甜点、饮品等
├── 菜品素材/            # 菜品描述参考
└── 笔记素材/            # 笔记文本素材
```

### 文件格式

每个素材包含两个文件：
- `菜名_001.jpg/png` - 图片文件
- `菜名_001.yaml` - 描述标签文件

### YAML标签内容

```yaml
filename: 琥珀牛肋_001.jpg
path: 素材库/图片素材库/肉类/

# 基础信息
primary_ingredient: 肉类
dish_type: 主菜
cuisine_style: 融合料理

# 视觉特征
main_colors:
  - 琥珀色
  - 棕色
composition: 俯拍
presentation_style: 左侧留白

# 详细描述
content_description: >
  菜品详细描述...

plating_details:
  plate: 纯白哑光圆形盘
  logic: 黄金分割构图
  spacing: 间距2-3cm
  decoration: 装饰说明
  negative_space: 左侧1/3留白
  overall_style: Bistro轻正餐
```

### 添加新素材

**方式1：手动添加**
1. 将图片放入对应分类目录
2. 创建同名 YAML 文件
3. 按规范填写标签内容

**方式2：批量迁移（推荐）**
1. 将图片放入 `参考图/` 目录
2. 运行命令：`/图片素材迁移`
3. 自动分析、命名、分类、生成YAML

## 使用规范

### 引用原则
1. **重新命名**：避免照搬素材库原菜名
2. **混合表达**：混合不同素材的表达方式
3. **改编创新**：改编而非复制

### 禁止行为
- ❌ 直接复制素材库原文
- ❌ 照搬素材库原菜名
- ❌ 连续使用相同结构
- ✅ 必须有至少1处创新

## 维护建议

1. **定期更新**：添加时令菜品图片
2. **质量检查**：确保YAML与图片一一对应
3. **命名规范**：使用「季节+颜色+食材+呈现」公式
4. **分类准确**：按主要食材放入正确分类
